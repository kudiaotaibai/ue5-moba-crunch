# 图形学基础补课 - 超小白版

## 先说结论

你现在看不懂，不是因为你笨，也不是因为 RDG 太难，而是因为你缺的是更前面的 3 层基础：

1. `屏幕上到底在画什么`
2. `为什么物体会前后遮挡`
3. `程序是怎么把“场景”变成“像素”的`

所以这篇不讲宏，不讲模板，不讲复杂数学，只讲最基本的直觉。

你现在只需要先搞懂这些词：

- 像素
- 颜色
- 深度
- 深度缓冲
- 纹理
- Shader
- 材质
- 屏幕空间效果

只要这几个词通了，后面的 RDG 才会开始有意义。

## 1. 什么是“图形学”

最简单的说法：

`图形学 = 想办法把场景变成屏幕上的一张图`

场景里本来有什么？

- 模型
- 灯光
- 摄像机
- 材质
- 天空

屏幕上最后有什么？

- 一堆像素点

所以图形学本质上就在做一件事：

`算出屏幕上每个像素应该是什么颜色`

## 2. 什么是像素

你可以把屏幕想成一张方格纸。

每个小格子就是一个像素。

比如一个像素可能是：

- 红色
- 蓝色
- 灰色
- 白色

如果很多像素排在一起，你就看到了一张完整画面。

所以图形程序最后总要回答一个问题：

`这个位置的像素，颜色是多少？`

## 3. 什么是“深度”

深度你先不要想得太数学。

最朴素的理解就是：

`这个东西离相机有多远`

比如：

- 你脸前面的杯子，深度小，因为离你近
- 窗外远处的山，深度大，因为离你远

所以：

- 近 = 深度小
- 远 = 深度大

这就是“深度”最基本的含义。

## 4. 为什么需要深度

因为会发生遮挡。

比如你站在屋里看窗外：

- 近处的窗框
- 远处的山

它们可能在屏幕上投影到同一个位置。

那这个位置最后显示谁？

答案当然是：

`显示更近的那个`

所以程序必须知道：

`谁更近，谁更远`

这就是深度存在的意义。

## 5. 什么是深度缓冲

深度缓冲可以先把它想成：

`一张专门记“距离”的表`

颜色缓冲记什么？

- 每个像素最后显示什么颜色

深度缓冲记什么？

- 每个像素当前已经画到的最近距离是多少

你可以想成屏幕上每个像素都有两个小本子：

- 一个本子记颜色
- 一个本子记距离

当新的物体要画到某个像素上时，程序会比较：

- 新来的这个点更近吗？

如果更近：

- 更新颜色
- 更新深度

如果更远：

- 不更新

所以深度缓冲的本质是：

`帮助 GPU 决定哪个物体应该挡住哪个物体`

## 6. 什么是深度图

深度图其实就是把“距离”显示出来。

因为深度本来只是数值，不是图。

为了让人看懂，我们通常把它变成灰度图：

- 近处更黑
- 远处更白

这样你一眼就能看出前后关系。

所以你现在做的这个插件，其实就是：

`把原本给机器看的深度信息，变成人类能看懂的黑白图`

## 7. 什么叫“线性深度”

这个词最容易把新手吓住。

你先只记一句：

`线性深度 = 更像真实距离的深度`

为什么要加“线性”两个字？

因为 GPU 深度缓冲里存的那个值，很多时候不是“真实距离本身”，而是一种更适合渲染计算的值。

这个值通常会让：

- 近处更精细
- 远处更压缩

所以深度缓冲里的原始值，不太适合直接拿来做人类看的灰度图。

于是我们会做一步转换：

`把深度缓冲里的值，转成更像真实距离的值`

这一步转换后的结果，就常被叫做：

`线性深度`

你现在先不用死记公式，只要知道：

- 原始深度值：更适合 GPU 算
- 线性深度：更适合人理解“离相机多远”

## 8. 什么是纹理

纹理最简单的理解就是：

`一张图片`

比如：

- 木头表面的图片
- 石头表面的图片
- 角色脸上的贴图
- UI 图标

这些都可以叫纹理。

但在图形学里，“纹理”不只可以存颜色，还可以存别的数据：

- 深度
- 法线
- 遮罩
- 速度

所以你不要把纹理只理解成“美术图片”。

更准确地说：

`纹理 = GPU 上一块按二维表存数据的资源`

只是最常见的情况是“它看起来像图片”。

## 9. 什么是采样纹理

采样纹理你先理解成：

`从这张图里读取某个位置的数据`

比如：

- 读颜色纹理，就是读取某点颜色
- 读深度纹理，就是读取某点深度

你现在这个 shader 里，本质上就是在做：

`从深度纹理里，读取当前像素位置的深度`

## 10. 什么是 Shader

Shader 可以先理解成：

`运行在 GPU 上的小程序`

它负责算图像。

比如 Shader 可以做：

- 算一个像素该是什么颜色
- 从纹理里读数据
- 做模糊
- 做发光
- 做深度可视化

你现在写的 `.usf` 文件，本质上就是 shader 代码。

## 11. 什么是像素着色器

像素着色器可以粗暴理解成：

`专门算“这个像素该显示什么颜色”的 shader`

你现在的 `MainPS` 做的事其实非常简单：

1. 找到当前像素位置
2. 读取这个位置的深度
3. 把深度变成灰度值
4. 输出这个灰度颜色

这就是一个非常标准的像素着色器思路。

## 12. 什么是材质

材质不是“图片”，也不是“shader 文件”本身。

你可以先把材质理解成：

`告诉引擎这个表面应该怎么显示的一套规则`

比如：

- 颜色是什么
- 粗糙度是多少
- 金属感是多少
- 法线怎么扰动

在 UE 里，材质往往是更高层的东西。

而你现在做的 RDG 插件，不是在写普通材质，而是在更底层地直接写全屏 shader pass。

所以现在你要先区分两个层次：

- 材质：更偏“给物体表面用”
- RDG/Global Shader：更偏“我自己插入一段渲染流程”

## 13. 什么是屏幕空间效果

屏幕空间效果可以理解成：

`不是去改场景里的模型，而是直接改“已经快画好的整张屏幕图”`

比如：

- 模糊
- 描边
- 泛光
- 深度可视化

这些效果通常都是：

- 画面已经快出来了
- 再对整张屏幕做一次处理

你现在这个深度灰度图，就是非常典型的屏幕空间效果。

因为它不是改某个模型，而是直接覆盖整个屏幕。

## 14. 你现在这个插件到底在做什么

把它讲成人话，大概就是：

1. UE 正常把场景渲染出来
2. 在后处理阶段，把你的代码插进去
3. 你的代码让 GPU 再跑一个全屏 pass
4. 这个 pass 不看原来的彩色画面，而是去读每个像素的深度
5. 再把深度变成黑白颜色输出
6. 所以最后你看到的是一张深度灰度图

如果再简化成一句话：

`你不是在画场景，你是在“重新解释”场景每个像素的深度`

## 15. 为什么你现在的画面一大片发白

因为你现在做的是：

```hlsl
DepthGray = saturate(LinearDepth / 10000.0f);
```

它的意思不是很复杂：

- 假设 `10000cm` 当成“白色上限”
- 比这个更远的，直接夹成 1，也就是白色

所以如果你的关卡很大：

- 远山
- 天空
- 地平线

这些位置很容易都变得接近白色。

这不是程序坏了，只是：

`你当前选的可视化范围不合适`

## 16. 点乘和叉乘你现在要不要补

要补，但不是现在最优先。

你当前阶段最优先的是：

1. 搞懂像素是什么
2. 搞懂深度是什么
3. 搞懂深度缓冲为什么能解决遮挡
4. 搞懂 shader 是在算像素颜色

点乘和叉乘更常出现在：

- 光照
- 法线
- 方向关系
- 反射

它们很重要，但不是你理解“深度灰度图”最先必须掌握的东西。

### 现在你只要先记一句

- 点乘：更像“两个方向有多一致”
- 叉乘：更像“从两个方向算出一个垂直方向”

这就够了。

## 17. 你现在最该按什么顺序补

### 第 1 层：先补直觉

先真正搞懂：

- 像素
- 深度
- 深度缓冲
- 纹理
- Shader

### 第 2 层：再看当前这版代码

只需要理解：

- C++ 把一个全屏 pass 加进后处理

### 第 3 层：再补一点数学

按这个顺序就行：

`深度 -> 法线 -> 模糊 -> SSAO`

## 18. 你现在完全不懂，能不能继续做

可以，完全可以。

而且你现在最正确的做法，不是去硬啃一整本图形学教材，而是：

`边做一个很小的效果，边补这个效果刚好需要的基础`

这就是我们为什么没直接上 SSAO，而是先做：

- 纯色全屏
- 深度灰度图

因为这两个东西正好可以逼着你把最基础的概念补起来。

## 19. 这篇看完后，你最少要记住哪几句话

如果你只记住下面这几句，就已经够用了：

- 图形学本质上是在算每个像素的颜色
- 深度就是离相机有多远
- 深度缓冲是用来解决遮挡的
- 纹理不只是图片，也可以存深度这类数据
- Shader 是跑在 GPU 上、负责算图像的小程序
- 你当前这个插件做的事，就是把深度变成一张黑白图

## 20. 一句话总结

你现在不要把目标定成“我马上彻底懂图形学”，而应该定成：

`我先把“深度是什么、shader 在干什么、这张灰度图为什么能出来”真正讲清楚`

做到这一步，你就已经不是完全小白了。
