## CGameInstance 类概览

### 什么是 GameInstance？

**通俗解释：**
- GameInstance 是整个游戏的"大管家"
- 它在游戏启动时创建，游戏关闭时销毁
- 负责管理跨关卡的数据和功能

**为什么网络功能放在这里？**
- ✅ 生命周期长：不会因为切换关卡而销毁
- ✅ 全局访问：任何地方都能访问到
- ✅ 单例模式：整个游戏只有一个实例

### 类的结构

**文件位置：** `Source/Crunch/Public/Framework/CGameInstance.h`

```cpp
class UCGameInstance : public UGameInstance

};
```

## 登录系统详解

### 为什么需要登录？

**问题：**
- 怎么知道你是谁？
- 怎么防止作弊？
- 怎么保存你的游戏数据？

**解决方案：**
- 使用 Epic 账号登录
- EOS 提供身份验证
- 每个玩家有唯一的 ID

### 代码详解：ClientAccountPortalLogin()

**文件位置：** `Source/Crunch/Private/Framework/CGameInstance.cpp:396`

```cpp
void UCGameInstance::ClientAccountPortalLogin()
{
    ClientLogin("AccountPortal", "", "");
}
```

**解释：**
- **函数名：** ClientAccountPortalLogin（客户端账号门户登录）
- **作用：** 启动 Epic 账号登录流程
- **"AccountPortal"：** 登录类型，表示使用 Epic 的账号门户
- **两个空字符串：** ID 和 Token，账号门户登录不需要这些

**调用流程：**
```
玩家点击"登录"按钮
    ↓
调用 ClientAccountPortalLogin()
    ↓
调用 ClientLogin("AccountPortal", "", "")
    ↓
弹出 Epic 登录窗口
    ↓
玩家输入账号密码
    ↓
登录成功！
```

### 代码详解：ClientLogin()

```cpp
void UCGameInstance::ClientLogin(const FString& Type, const FString& Id, const FString& Token)
{
    // 1. 获取 EOS 身份接口
    if (IOnlineIdentityPtr IdentityPtr = UCNetStatics::GetIdentityPtr())
    {
        // 2. 如果之前有登录回调，先移除
        if (LoggingInDelegateHandle.IsValid())
        {
            IdentityPtr->OnLoginCompleteDelegates->Remove(LoggingInDelegateHandle);
            LoggingInDelegateHandle.Reset();
        }
        
        // 3. 注册登录完成回调
        LoggingInDelegateHandle = IdentityPtr->OnLoginCompleteDelegates->AddUObject(
            this, &UCGameInstance::LoginCompleted
        );
        
        // 4. 执行登录
        if (!IdentityPtr->Login(0, FOnlineAccountCredentials(Type, Id, Token)))
        {
            // 登录失败，清理回调
            UE_LOG(LogTemp, Warning, TEXT("Login failed!"));
            if (LoggingInDelegateHandle.IsValid())
            {
                IdentityPtr->OnLoginCompleteDelegates->Remove(LoggingInDelegateHandle);
                LoggingInDelegateHandle.Reset();
            }
            OnLoginCompleted.Broadcast(false, "", "Login failed! Right Away!");
        }
    }
}
```

**逐段解释：**

#### 第1步：获取身份接口

```cpp
if (IOnlineIdentityPtr IdentityPtr = UCNetStatics::GetIdentityPtr())
```

- **IOnlineIdentityPtr：** EOS 身份接口的智能指针
- **GetIdentityPtr()：** 获取 EOS 的身份管理接口
- **作用：** 这个接口负责所有登录相关的操作
- **通俗解释：** 就像拿到了"登录管理员"的电话号码

#### 第2步：清理旧的回调

```cpp
if (LoggingInDelegateHandle.IsValid())
{
    IdentityPtr->OnLoginCompleteDelegates->Remove(LoggingInDelegateHandle);
    LoggingInDelegateHandle.Reset();
}
```

- **LoggingInDelegateHandle：** 回调句柄，用来标识一个回调函数
- **IsValid()：** 检查句柄是否有效
- **Remove()：** 移除之前注册的回调
- **Reset()：** 重置句柄
- **为什么要这样做：** 防止重复注册回调，避免内存泄漏

**通俗解释：**
```
就像你订了外卖，但又取消了
如果不取消，外卖员还会送来
这里是取消之前的"登录完成通知"
```

#### 第3步：注册新的回调

```cpp
LoggingInDelegateHandle = IdentityPtr->OnLoginCompleteDelegates->AddUObject(
    this, &UCGameInstance::LoginCompleted
);
```

- **OnLoginCompleteDelegates：** 登录完成事件的委托列表
- **AddUObject()：** 添加一个 UObject 的成员函数作为回调
- **this：** 当前对象（UCGameInstance）
- **&UCGameInstance::LoginCompleted：** 回调函数的地址

**通俗解释：**

```
告诉 EOS："登录完成后，请调用我的 LoginCompleted() 函数"
就像留个电话号码，让快递员送到了给你打电话
```

#### 第4步：执行登录

```cpp
if (!IdentityPtr->Login(0, FOnlineAccountCredentials(Type, Id, Token)))
{
    // 登录失败处理...
}
```

- **Login()：** 执行登录操作
- **参数1：** `0` - 本地玩家索引（第一个玩家）
- **参数2：** `FOnlineAccountCredentials` - 登录凭证
  - **Type：** "AccountPortal"（登录类型）
  - **Id：** 用户ID（账号门户登录时为空）
  - **Token：** 令牌（账号门户登录时为空）
- **返回值：** `false` 表示登录请求失败（不是登录失败，是请求都没发出去）

**通俗解释：**

```
Type = "AccountPortal" 就像说："我要用 Epic 账号登录"
如果 Type = "Password"，就需要提供 Id 和 Token（用户名和密码）
```

### 代码详解：LoginCompleted()

```cpp
void UCGameInstance::LoginCompleted(
    int NumOfLocalPlayer,
    bool bWasSuccessful,
    const FUniqueNetId& UserId,
    const FString& Error
)
{
    // 1. 获取身份接口
    if (IOnlineIdentityPtr IdentityPtr = UCNetStatics::GetIdentityPtr())
    {
        // 2. 移除回调
        if (LoggingInDelegateHandle.IsValid())
        {
            IdentityPtr->OnLoginCompleteDelegates->Remove(LoggingInDelegateHandle);
            LoggingInDelegateHandle.Reset();
        }

        // 3. 处理登录结果
        FString PlayerNickname = "";
        if (bWasSuccessful)
        {
            // 登录成功，获取玩家昵称
            PlayerNickname = IdentityPtr->GetPlayerNickname(UserId);
            UE_LOG(LogTemp, Warning, TEXT("Login successfully PlayerNickname: %s"), *PlayerNickname);
        }
        else
        {
            // 登录失败，记录错误
            UE_LOG(LogTemp, Warning, TEXT("Login failed! :%s"), *Error);
        }
        
        // 4. 广播登录完成事件
        OnLoginCompleted.Broadcast(bWasSuccessful, PlayerNickname, Error);
    }
    else
    {
        // 找不到身份接口
        OnLoginCompleted.Broadcast(false, "", "Can't find Identity Pointer");
    }
}
```

**参数解释：**

```cpp
int NumOfLocalPlayer
```
- **作用：** 本地玩家数量
- **通常是：** 0（第一个玩家）
- **用途：** 支持分屏多人游戏

```cpp
bool bWasSuccessful
```
- **作用：** 登录是否成功
- **true：** 登录成功
- **false：** 登录失败

```cpp
const FUniqueNetId& UserId
```
- **作用：** 玩家的唯一ID
- **类型：** FUniqueNetId（EOS 提供的唯一标识）
- **用途：** 用来识别玩家身份

```cpp
const FString& Error
```

- **作用：** 错误信息
- **登录成功时：** 为空
- **登录失败时：** 包含失败原因

**流程图：**

```
LoginCompleted() 被调用
    ↓
检查登录是否成功？
    ├─ 成功 → 获取玩家昵称
    │         ↓
    │    记录日志："登录成功，昵称：XXX"
    │         ↓
    │    广播事件：OnLoginCompleted(true, "XXX", "")
    │
    └─ 失败 → 记录日志："登录失败：XXX"
              ↓
         广播事件：OnLoginCompleted(false, "", "错误信息")
```

### 登录状态查询

```cpp
bool UCGameInstance::IsLoggedIn() const
{
    if (IOnlineIdentityPtr IdentityPtr = UCNetStatics::GetIdentityPtr())
    {
        return IdentityPtr->GetLoginStatus(0) == ELoginStatus::LoggedIn;
    }
    return false;
}
```

**解释：**
- **GetLoginStatus(0)：** 获取第一个玩家的登录状态
- **ELoginStatus::LoggedIn：** 已登录状态
- **返回值：** true = 已登录，false = 未登录

```cpp
bool UCGameInstance::IsLoggingIn() const
{
    return LoggingInDelegateHandle.IsValid();
}
```

**解释：**
- **作用：** 检查是否正在登录中
- **LoggingInDelegateHandle.IsValid()：** 如果句柄有效，说明正在等待登录回调
- **用途：** 防止重复登录

## 创建游戏会话

### 什么是"创建会话"？

**通俗解释：**
```
创建会话 = 开房间
就像你在网吧开了一个游戏房间，等朋友进来一起玩
```

**流程：**
```
1. 玩家点击"创建游戏"
2. 客户端向协调器发送 HTTP 请求
3. 协调器启动游戏服务器
4. 服务器创建 EOS 会话
5. 客户端搜索并加入会话
6. 开始游戏！
```

### 代码详解：RequestCreateAndJoinSession()

**文件位置：** `Source/Crunch/Private/Framework/CGameInstance.cpp:36`

```cpp
void UCGameInstance::RequestCreateAndJoinSession(const FName& NewSessionName)
{
    // 1. 记录日志
    UE_LOG(LogTemp, Warning, TEXT("Request Create And Join Session:%s"), *(NewSessionName.ToString()));
    
    // 2. 创建 HTTP 请求
    FHttpRequestRef Request = FHttpModule::Get().CreateRequest();
    
    // 3. 生成会话搜索ID
    FGuid SessionSearchId = FGuid::NewGuid();

    // 4. 获取协调器 URL
    FString CoordinatorURL = UCNetStatics::GetCoordinatorURL();
    FString URL = FString::Printf(TEXT("%s/Sessions"), *CoordinatorURL);
    UE_LOG(LogTemp, Warning, TEXT("Send Request Session Creation to URL: %s"), *URL);

    // 5. 设置请求参数
    Request->SetURL(URL);
    Request->SetVerb("POST");
    Request->SetHeader(TEXT("Content-Type"), TEXT("application/json"));

    // 6. 构建 JSON 请求体
    TSharedPtr<FJsonObject> JsonObject = MakeShareable(new FJsonObject());
    JsonObject->SetStringField(UCNetStatics::GetSessionNameKey().ToString(), NewSessionName.ToString());
    JsonObject->SetStringField(UCNetStatics::GetSessionSearchIdKey().ToString(), SessionSearchId.ToString());

    FString RequestBody;
    TSharedRef<TJsonWriter<>> Writer = TJsonWriterFactory<>::Create(&RequestBody);
    FJsonSerializer::Serialize(JsonObject.ToSharedRef(), Writer);

    // 7. 设置请求体
    Request->SetContentAsString(RequestBody);
    
    // 8. 绑定回调函数
    Request->OnProcessRequestComplete().BindUObject(
        this, &UCGameInstance::SessionCreationRequestCompleted, SessionSearchId
    );

    // 9. 发送请求
    if (!Request->ProcessRequest())
    {
        UE_LOG(LogTemp, Warning, TEXT("Request ProcessRequest Failed"));
    }
}
```

**逐段详解：**

#### 第1步：记录日志

```cpp
UE_LOG(LogTemp, Warning, TEXT("Request Create And Join Session:%s"), *(NewSessionName.ToString()));
```

- **UE_LOG：** Unreal 的日志宏
- **LogTemp：** 日志类别（临时日志）
- **Warning：** 日志级别（警告）
- **TEXT()：** 宏，用于跨平台字符串
- **作用：** 在控制台输出："请求创建并加入会话：XXX"

#### 第2步：创建 HTTP 请求

```cpp
FHttpRequestRef Request = FHttpModule::Get().CreateRequest();
```

- **FHttpModule::Get()：** 获取 HTTP 模块单例
- **CreateRequest()：** 创建一个 HTTP 请求对象
- **FHttpRequestRef：** HTTP 请求的智能指针（引用计数）
- **通俗解释：** 就像拿了一张表单，准备填写并发送

#### 第3步：生成会话搜索ID

```cpp
FGuid SessionSearchId = FGuid::NewGuid();
```

- **FGuid：** 全局唯一标识符（Globally Unique Identifier）
- **NewGuid()：** 生成一个新的 GUID
- **格式：** `123e4567-e89b-12d3-a456-426614174000`
- **作用：** 给这个会话一个唯一的ID，方便后续搜索

**为什么需要这个ID？**
```
假设有10个房间都叫"赵云飞的房间"
怎么区分？用 SessionSearchId！
每个房间的 ID 都不一样
```

#### 第4步：构建 URL

```cpp
FString CoordinatorURL = UCNetStatics::GetCoordinatorURL();
FString URL = FString::Printf(TEXT("%s/Sessions"), *CoordinatorURL);
```

- **GetCoordinatorURL()：** 获取协调器的地址
  - 可能是：`http://localhost:7777`
  - 或者：`http://192.168.1.100:7777`
- **FString::Printf()：** 格式化字符串
- **最终 URL：** `http://localhost:7777/Sessions`

#### 第5步：设置请求参数

```cpp
Request->SetURL(URL);
Request->SetVerb("POST");
Request->SetHeader(TEXT("Content-Type"), TEXT("application/json"));
```

- **SetURL()：** 设置请求地址
- **SetVerb("POST")：** 设置 HTTP 方法为 POST
  - GET：获取数据
  - POST：提交数据
  - PUT：更新数据
  - DELETE：删除数据
- **SetHeader()：** 设置请求头
  - `Content-Type: application/json` 表示发送 JSON 数据

#### 第6步：构建 JSON 请求体

```cpp
TSharedPtr<FJsonObject> JsonObject = MakeShareable(new FJsonObject());
JsonObject->SetStringField(UCNetStatics::GetSessionNameKey().ToString(), NewSessionName.ToString());
JsonObject->SetStringField(UCNetStatics::GetSessionSearchIdKey().ToString(), SessionSearchId.ToString());
```

- **TSharedPtr<FJsonObject>：** JSON 对象的智能指针
- **MakeShareable()：** 创建智能指针
- **SetStringField()：** 设置字符串字段

**生成的 JSON：**
```json
{
  "SESSION_NAME": "赵云飞的房间",
  "SESSION_SEARCH_ID": "123e4567-e89b-12d3-a456-426614174000"
}
```

```cpp
FString RequestBody;
TSharedRef<TJsonWriter<>> Writer = TJsonWriterFactory<>::Create(&RequestBody);
FJsonSerializer::Serialize(JsonObject.ToSharedRef(), Writer);
```

- **TJsonWriter：** JSON 写入器
- **TJsonWriterFactory::Create()：** 创建写入器
- **FJsonSerializer::Serialize()：** 把 JSON 对象序列化成字符串
- **结果：** RequestBody 包含 JSON 字符串

#### 第7步：设置请求体

```cpp
Request->SetContentAsString(RequestBody);
```

- **作用：** 把 JSON 字符串设置为请求体
- **相当于：** 把表单内容填好

#### 第8步：绑定回调

```cpp
Request->OnProcessRequestComplete().BindUObject(
    this, &UCGameInstance::SessionCreationRequestCompleted, SessionSearchId
);
```

- **OnProcessRequestComplete()：** 请求完成事件
- **BindUObject()：** 绑定 UObject 的成员函数
- **参数：**
  - `this`：当前对象
  - `&UCGameInstance::SessionCreationRequestCompleted`：回调函数
  - `SessionSearchId`：额外参数，传递给回调函数

**通俗解释：**
```
告诉 HTTP 模块："请求完成后，调用我的 SessionCreationRequestCompleted() 函数"
并且把 SessionSearchId 传过去
```

#### 第9步：发送请求

```cpp
if (!Request->ProcessRequest())
{
    UE_LOG(LogTemp, Warning, TEXT("Request ProcessRequest Failed"));
}
```

- **ProcessRequest()：** 发送 HTTP 请求
- **返回值：** `false` 表示请求发送失败
- **注意：** 这里的失败是指请求没发出去，不是服务器返回失败
