## CGameSession 详解

### 什么是 GameSession？

**通俗解释：**
```
GameSession = 游戏会话管理器
就像一个"前台接待员"
负责处理玩家的进出
```

**主要职责：**
- 玩家连接时注册
- 玩家断开时注销
- 管理玩家列表

### 代码详解：RegisterPlayer()

**文件位置：** `Source/Crunch/Private/Network/CGameSession.cpp:13`

```cpp
void ACGameSession::RegisterPlayer(
    APlayerController* NewPlayer,
    const FUniqueNetIdRepl& UniqueId,
    bool bWasFromInvite
)
{
    // 1. 调用父类方法（必须）
    Super::RegisterPlayer(NewPlayer, UniqueId, bWasFromInvite);
    
    // 2. 通知 GameInstance
    if (UCGameInstance* GameInst = GetGameInstance<UCGameInstance>())
    {
        GameInst->PlayerJoined(UniqueId);
    }
}
```

**参数详解：**

```cpp
APlayerController* NewPlayer
```
- **类型：** 玩家控制器指针
- **作用：** 代表一个玩家
- **包含：** 玩家的输入、视角、拥有的Pawn等              

```cpp
const FUniqueNetIdRepl& UniqueId
```
- **类型：** 可复制的唯一网络ID
- **作用：** 玩家的唯一标识（来自 EOS）
- **用途：** 区分不同的玩家

```cpp
bool bWasFromInvite
```
- **类型：** 布尔值
- **作用：** 玩家是否通过邀请加入
- **true：** 通过好友邀请
- **false：** 通过搜索加入

**流程图：**

```
玩家连接到服务器
    ↓
Unreal 引擎调用 RegisterPlayer()
    ↓
调用父类方法（处理基础注册）
    ↓
通知 GameInstance（自定义逻辑）
    ↓
GameInstance::PlayerJoined()
    ├─ 清除超时定时器
    └─ 记录玩家ID
```

### 代码详解：UnregisterPlayer()

```cpp

**参数：**

```
- **类型：** 会话名称

- **举例：** "赵云飞的房间"

**流程：**

```
玩家断开连接
    ↓
Unreal 引擎调用 UnregisterPlayer()
    ↓
调用父类方法（处理基础注销）
    ↓
通知 GameInstance
    ↓
GameInstance::PlayerLeft()
    ├─ 移除玩家记录
    └─ 检查是否所有玩家都离开
```

## 网络同步基础

### 客户端-服务器模型

**架构：**

```
客户端1                    服务器                    客户端2
  ↓                         ↓                         ↓
玩家按下技能键          收到客户端1的请求          等待服务器通知
  ↓                         ↓                         ↓
发送 RPC 到服务器       验证并执行技能             收到服务器的复制
  ↓                         ↓                         ↓
等待服务器响应          同步给所有客户端           显示技能效果
  ↓                         ↓                         ↓
收到服务器的复制        更新游戏状态               游戏继续
  ↓                         ↓                         ↓
显示技能效果            游戏继续                   游戏继续
```

**核心原则：**

1. **服务器权威**
   - 所有重要逻辑在服务器执行
   - 客户端只负责表现
   - 防止作弊

2. **客户端预测**
   - 客户端先显示效果
   - 等服务器确认
   - 如果不一致，回滚

3. **状态同步**
   - 服务器定期同步状态
   - 客户端接收并更新
   - 保证所有玩家看到的一致

### 网络模式判断

**在代码中判断当前是客户端还是服务器：**

```cpp
// 方法1：使用 GetNetMode()
ENetMode NetMode = GetWorld()->GetNetMode();
if (NetMode == NM_DedicatedServer)
{
    // 这是服务器
}
else if (NetMode == NM_Client)
{
    // 这是客户端
}

// 方法2：使用 HasAuthority()
if (HasAuthority())
{
    // 这是服务器（有权威）
}
else
{
    // 这是客户端（无权威）
}

// 方法3：使用 IsLocallyControlled()
if (IsLocallyControlled())
{
    // 这是本地控制的（可能是客户端或监听服务器）
}
```

**通俗解释：**
```
HasAuthority() = 你是裁判吗？
  true：是裁判（服务器），可以做决定
  false：不是裁判（客户端），只能看

IsLocallyControlled() = 这是你控制的角色吗？
  true：是你的角色
  false：是别人的角色
```

## 属性复制系统

### 什么是属性复制？

**通俗解释：**
```
属性复制 = 自动同步数据
服务器的数据变化后，自动发送给客户端
客户端收到后，自动更新
```

### 实战案例：角色属性复制

**文件位置：** `Source/Crunch/Public/Character/CCharacter.h`

```cpp
class ACCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    // 声明需要复制的属性
    virtual void GetLifetimeReplicatedProps(
        TArray<class FLifetimeProperty>& OutLifetimeProps
    ) const override;

private:
    // 复制的属性
    UPROPERTY(ReplicatedUsing = OnRep_TeamID)
    FGenericTeamId TeamID;

    // 复制回调函数
    UFUNCTION()
    virtual void OnRep_TeamID();
};
```

**实现文件：**

```cpp
void ACCharacter::GetLifetimeReplicatedProps(
    TArray<FLifetimeProperty>& OutLifetimeProps
) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);
    
    // 注册需要复制的属性
    DOREPLIFETIME(ACCharacter, TeamID);doreplifetime
}

void ACCharacter::OnRep_TeamID()
{
    // 当 TeamID 在客户端更新时调用
    UE_LOG(LogTemp, Warning, TEXT("TeamID changed to: %d"), TeamID.GetId());
    
    // 更新角色外观等
    UpdateTeamVisuals();
}
```

**关键宏详解：**

```cpp
UPROPERTY(ReplicatedUsing = OnRep_TeamID)
```
- **Replicated：** 标记为需要复制的属性
- **ReplicatedUsing：** 指定复制回调函数
- **OnRep_TeamID：** 客户端收到更新时调用的函数

```cpp
DOREPLIFETIME(ACCharacter, TeamID);
```
- **作用：** 注册属性到复制系统
- **参数1：** 类名
- **参数2：** 属性名

**工作流程：**

```
服务器端：
    ↓
TeamID = 1（设置队伍ID）
    ↓
Unreal 检测到 TeamID 变化
    ↓
自动发送给所有客户端
    ↓
客户端：
    ↓
收到 TeamID = 1
    ↓
自动更新本地 TeamID
    ↓
调用 OnRep_TeamID()
    ↓
更新角色外观（比如换队服颜色）
```

### 复制条件

**DOREPLIFETIME 的变体：**

```cpp
// 1. 无条件复制（默认）
DOREPLIFETIME(ACCharacter, TeamID);

// 2. 只复制给拥有者
DOREPLIFETIME_CONDITION(ACCharacter, TeamID, COND_OwnerOnly);

// 3. 跳过拥有者（复制给其他人）
DOREPLIFETIME_CONDITION(ACCharacter, TeamID, COND_SkipOwner);

// 4. 初始化时复制一次
DOREPLIFETIME_CONDITION(ACCharacter, TeamID, COND_InitialOnly);

// 5. 自定义条件
DOREPLIFETIME_CONDITION(ACCharacter, TeamID, COND_Custom);
```

**举例：**

```cpp
// 玩家的金币数量
DOREPLIFETIME_CONDITION(AMyCharacter, Gold, COND_OwnerOnly);
// 只复制给拥有这个角色的玩家
// 其他玩家看不到你有多少钱

// 角色的位置
DOREPLIFETIME(AMyCharacter, Location);
// 复制给所有人
// 大家都能看到你在哪里
```

## RPC 远程调用

### 什么是 RPC？

**RPC = Remote Procedure Call（远程过程调用）**

**通俗解释：**
```
RPC 就像打电话
客户端："喂，服务器，我要释放技能！"
服务器："好的，我帮你执行"
```

### RPC 的三种类型

#### 1. Server RPC（客户端 → 服务器）

```cpp
// 声明
UFUNCTION(Server, Reliable, WithValidation)
void Server_UseSkill(int32 SkillID);

// 实现
void AMyCharacter::Server_UseSkill_Implementation(int32 SkillID)
{
    // 这段代码在服务器上执行
    if (HasAuthority())
    {
        UE_LOG(LogTemp, Warning, TEXT("Server: Using skill %d"), SkillID);
        // 执行技能逻辑
        CastSkill(SkillID);
    }
}

// 验证（防作弊）
bool AMyCharacter::Server_UseSkill_Validate(int32 SkillID)
{
    // 验证参数是否合法
    return SkillID >= 0 && SkillID < 10;
}

// 调用（在客户端）
void AMyCharacter::OnSkillButtonPressed(int32 SkillID)
{
    // 客户端调用，服务器执行
    Server_UseSkill(SkillID);
}
```

**关键字解释：**

```cpp
UFUNCTION(Server, Reliable, WithValidation)
```
- **Server：** 这是一个服务器 RPC
- **Reliable：** 可靠传输（保证送达）
  - 对应的是 **Unreliable**（不可靠，可能丢包）
- **WithValidation：** 需要验证函数

**命名规则：**
```
声明：Server_UseSkill
实现：Server_UseSkill_Implementation
验证：Server_UseSkill_Validate
```

#### 2. Client RPC（服务器 → 客户端）

```cpp
// 声明
UFUNCTION(Client, Reliable)
void Client_ShowDamageNumber(int32 Damage);

// 实现
void AMyCharacter::Client_ShowDamageNumber_Implementation(int32 Damage)
{
    // 这段代码在客户端执行
    UE_LOG(LogTemp, Warning, TEXT("Client: Show damage %d"), Damage);
    // 显示伤害数字UI
    ShowDamageUI(Damage);
}

// 调用（在服务器）
void AMyCharacter::TakeDamage(int32 Damage)
{
    if (HasAuthority())
    {
        // 服务器计算伤害
        Health -= Damage;
        
        // 通知客户端显示伤害数字
        Client_ShowDamageNumber(Damage);
    }
}
```

**特点：**
- 只发送给拥有这个 Actor 的客户端
- 其他客户端收不到

#### 3. Multicast RPC（服务器 → 所有客户端）

```cpp
// 声明
UFUNCTION(NetMulticast, Reliable)
void Multicast_PlaySkillEffect(FVector Location);

// 实现
void AMyCharacter::Multicast_PlaySkillEffect_Implementation(FVector Location)
{
    // 这段代码在所有客户端和服务器上执行
    UE_LOG(LogTemp, Warning, TEXT("Playing skill effect at %s"), *Location.ToString());
    // 播放技能特效
    SpawnEffectAtLocation(Location);
}

// 调用（在服务器）
void AMyCharacter::CastSkill()
{
    if (HasAuthority())
    {
        // 服务器执行技能逻辑
        FVector SkillLocation = GetActorLocation();
        
        // 通知所有客户端播放特效
        Multicast_PlaySkillEffect(SkillLocation);
    }
}
```

**特点：**
- 发送给所有客户端
- 服务器自己也会执行

### RPC 使用场景

| RPC 类型 | 调用方 | 执行方 | 使用场景 |
|---------|--------|--------|----------|
| Server | 客户端 | 服务器 | 玩家输入、请求执行操作 |
| Client | 服务器 | 单个客户端 | 通知特定玩家（UI更新、私密信息） |
| Multicast | 服务器 | 所有客户端 | 播放特效、音效、所有人都能看到的事件 |

## 实战案例

### 案例1：技能释放完整流程

**场景：** 玩家按下技能键，释放一个地面冲击技能

```cpp
// 1. 客户端：玩家按下技能键
void AMyCharacter::OnSkillButtonPressed()
{
    // 客户端预测：立即播放动画
    PlaySkillAnimation();
    
    // 发送到服务器
    Server_RequestCastSkill();
}

// 2. 服务器：验证并执行技能
void AMyCharacter::Server_RequestCastSkill_Implementation()
{
    if (!HasAuthority()) return;
    
    // 验证：检查法力值、冷却时间等
    if (Mana < 50)
    {
        // 法力不足，通知客户端
        Client_SkillFailed(TEXT("Not enough mana"));
        return;
    }
    
    // 扣除法力值
    Mana -= 50;
    
    // 计算伤害
    TArray<AActor*> HitActors = GetActorsInRange(500.f);
    for (AActor* Actor : HitActors)
    {
        if (AMyCharacter* Enemy = Cast<AMyCharacter>(Actor))
        {
            // 造成伤害
            Enemy->TakeDamage(100);
        }
    }
    
    // 通知所有客户端播放特效
    Multicast_PlaySkillEffect(GetActorLocation());
}

// 3. 所有客户端：播放特效
void AMyCharacter::Multicast_PlaySkillEffect_Implementation(FVector Location)
{
    // 播放粒子特效
    UGameplayStatics::SpawnEmitterAtLocation(
        GetWorld(),
        SkillEffectTemplate,
        Location
    );
    
    // 播放音效
    UGameplayStatics::PlaySoundAtLocation(
        GetWorld(),
        SkillSound,
        Location
    );
}

// 4. 客户端：技能失败处理
void AMyCharacter::Client_SkillFailed_Implementation(const FString& Reason)
{
    // 显示错误提示
    ShowErrorMessage(Reason);
    
    // 停止动画
    StopSkillAnimation();
}
```

**流程图：**

```
客户端1（释放技能的玩家）
    ↓
按下技能键
    ↓
立即播放动画（客户端预测）
    ↓
发送 Server_RequestCastSkill()
    ↓
服务器
    ↓
验证法力值、冷却时间
    ├─ 失败 → Client_SkillFailed() → 客户端1显示错误
    └─ 成功 → 扣除法力值
              ↓
         计算伤害，更新敌人血量
              ↓
         Multicast_PlaySkillEffect()
              ↓
         ┌────┴────┬────────┐
         ↓         ↓        ↓
    客户端1    客户端2   服务器
         ↓         ↓        ↓
    播放特效  播放特效  播放特效
```

### 案例2：角色属性同步

**场景：** 角色升级，等级和属性同步给所有客户端

```cpp
// 角色类
class AMyCharacter : public ACharacter
{
    UPROPERTY(ReplicatedUsing = OnRep_Level)
    int32 Level;
    
    UPROPERTY(ReplicatedUsing = OnRep_MaxHealth)
    float MaxHealth;
    
    UFUNCTION()
    void OnRep_Level();
    
    UFUNCTION()
    void OnRep_MaxHealth();
    
    UFUNCTION(Server, Reliable)
    void Server_LevelUp();
};

// 服务器：升级逻辑
void AMyCharacter::Server_LevelUp_Implementation()
{
    if (!HasAuthority()) return;
    
    // 升级
    Level++;
    MaxHealth += 100;
    
    // 这些属性会自动复制给所有客户端
    UE_LOG(LogTemp, Warning, TEXT("Server: Level up to %d"), Level);
}

// 客户端：收到等级更新
void AMyCharacter::OnRep_Level()
{
    UE_LOG(LogTemp, Warning, TEXT("Client: Level changed to %d"), Level);
    
    // 更新UI
    UpdateLevelUI(Level);
    
    // 播放升级特效
    PlayLevelUpEffect();
}

// 客户端：收到最大生命值更新
void AMyCharacter::OnRep_MaxHealth()
{
    UE_LOG(LogTemp, Warning, TEXT("Client: MaxHealth changed to %f"), MaxHealth);
    
    // 更新UI
    UpdateHealthBar();
}
```

## 网络优化技巧

### 1. 减少 RPC 调用频率

**❌ 不好的做法：**
```cpp
void AMyCharacter::Tick(float DeltaTime)
{
    // 每帧都发送 RPC（60次/秒）
    Server_UpdatePosition(GetActorLocation());
}
```

**✅ 好的做法：**
```cpp
void AMyCharacter::Tick(float DeltaTime)
{
    // 使用属性复制，引擎会自动优化频率
    // 或者限制发送频率
    TimeSinceLastUpdate += DeltaTime;
    if (TimeSinceLastUpdate >= 0.1f)  // 每0.1秒发送一次
    {
        Server_UpdatePosition(GetActorLocation());
        TimeSinceLastUpdate = 0.f;
    }
}
```

### 2. 使用 Unreliable RPC

**对于不重要的数据，使用 Unreliable：**

```cpp
// 重要数据：使用 Reliable
UFUNCTION(Server, Reliable)
void Server_PurchaseItem(int32 ItemID);

// 不重要数据：使用 Unreliable
UFUNCTION(NetMulticast, Unreliable)
void Multicast_PlayFootstepSound();
```

**区别：**
- **Reliable：** 保证送达，但可能延迟
- **Unreliable：** 可能丢包，但延迟低

### 3. 批量发送数据

**❌ 不好的做法：**
```cpp
for (int32 i = 0; i < 100; i++)
{
    Server_AddItem(i);  // 发送100次 RPC
}
```

**✅ 好的做法：**
```cpp
TArray<int32> Items;
for (int32 i = 0; i < 100; i++)
{
    Items.Add(i);
}
Server_AddItems(Items);  // 发送1次 RPC
```

## 总结

### 网络系统核心要点

1. **服务器权威**
   - 重要逻辑在服务器执行
   - 客户端只负责表现
   - 防止作弊

2. **属性复制**
   - 自动同步数据
   - 使用 UPROPERTY(Replicated)
   - 回调函数处理更新

3. **RPC 调用**
   - Server RPC：客户端 → 服务器
   - Client RPC：服务器 → 单个客户端
   - Multicast RPC：服务器 → 所有客户端

4. **性能优化**
   - 减少 RPC 频率
   - 使用 Unreliable 传输不重要数据
   - 批量发送数据
