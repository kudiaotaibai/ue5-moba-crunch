## 什么是单例模式

### 基本概念

**单例模式（Singleton Pattern）**：确保一个类只有一个实例，并提供全局访问点。

**通俗理解：**
就像一个国家只能有一个总统，一个游戏只能有一个游戏管理器。无论你在哪里，都能找到这个唯一的实例。

### 单例模式的特点

1. **唯一性**：整个程序中只有一个实例
2. **全局访问**：任何地方都能访问到这个实例
3. **延迟创建**：通常在第一次使用时才创建
4. **自我管理**：类自己负责创建和管理实例

### 基本结构

```cpp
class Singleton {
private:
    // 私有构造函数，防止外部创建实例
    Singleton() {}
    
    // 私有拷贝构造函数，防止复制
    Singleton(const Singleton&) = delete;
    
    // 私有赋值操作符，防止赋值
    Singleton& operator=(const Singleton&) = delete;
    
public:
    // 获取唯一实例的静态方法
    static Singleton* GetInstance();
    
    // 业务方法
    void DoSomething();
};
```

**代码解释：**
- `private:`：私有成员，外部无法直接访问
- `Singleton() {}`：构造函数，创建对象时调用
- `= delete`：C++11 语法，明确禁用某个函数
- `static`：静态成员，属于类而不是对象
- `GetInstance()`：获取实例的方法名（约定俗成）

## 为什么需要单例模式

### 使用场景

**1. 资源管理器**
```cpp
// 游戏中的资源管理器，全局只需要一个
class ResourceManager {
public:
    void LoadTexture(const std::string& path);
    void LoadSound(const std::string& path);
};
```

**2. 配置管理器**
```cpp
// 游戏配置，全局共享
class ConfigManager {
public:
    int GetScreenWidth();
    int GetScreenHeight();
    float GetMasterVolume();
};
```

**3. 日志系统**
```cpp
// 日志记录，全局统一
class Logger {
public:
    void Log(const std::string& message);
    void Error(const std::string& error);
};
```

### 优点

1. **节省内存**：只有一个实例
2. **全局访问**：任何地方都能使用
3. **延迟初始化**：需要时才创建
4. **数据一致性**：所有地方使用同一份数据

### 缺点

1. **违反单一职责**：类既要管理自己的实例，又要处理业务逻辑
2. **难以测试**：全局状态难以模拟
3. **隐藏依赖**：不容易看出类之间的依赖关系
4. **线程安全问题**：多线程环境下需要特殊处理

## 线程安全基础知识

### 什么是线程？

**线程（Thread）**：程序执行的最小单位，一个进程可以有多个线程。

**通俗理解：**
就像一个工厂（进程）里有多个工人（线程），他们可以同时工作，但需要协调避免冲突。

### 什么是线程安全问题？

**问题场景：**
```cpp
// 假设两个线程同时调用 GetInstance()
Thread 1: if (instance == nullptr)  // 检查：instance 是 nullptr
Thread 2: if (instance == nullptr)  // 检查：instance 还是 nullptr
Thread 1: instance = new Singleton(); // 创建实例 A
Thread 2: instance = new Singleton(); // 创建实例 B，覆盖了 A
```

**结果：**
- 创建了两个实例（违反单例原则）
- 实例 A 内存泄漏
- 程序可能崩溃

**通俗理解：**
两个人同时看到冰箱里没有牛奶，都去买牛奶，结果买了两瓶。

### 竞态条件（Race Condition）

**定义：**
多个线程访问共享资源时，结果依赖于线程执行的时序。

**示例：**
```cpp
int counter = 0;

// 线程 1
counter++;  // 读取 counter(0) -> 加1 -> 写入 counter(1)

// 线程 2  
counter++;  // 读取 counter(0) -> 加1 -> 写入 counter(1)

// 期望结果：counter = 2
// 实际结果：counter = 1（因为两个线程读到了同样的初始值）
```

### 互斥锁（Mutex）

**Mutex = Mutual Exclusion（互斥）**

**作用：**
确保同一时间只有一个线程可以访问共享资源。

**通俗理解：**
就像厕所门锁，一次只能一个人进去，其他人要等待。

**基本用法：**
```cpp
#include <mutex>

std::mutex mtx;  // 创建互斥锁

void SafeFunction() {
    mtx.lock();      // 加锁（如果已被锁，当前线程会等待）
    // 临界区代码（只有一个线程能执行）
    // ...
    mtx.unlock();    // 解锁
}
```

**函数解释：**
- `std::mutex mtx`：创建一个互斥锁对象
- `mtx.lock()`：获取锁，如果锁被其他线程占用，当前线程会阻塞等待
- `mtx.unlock()`：释放锁，让其他等待的线程可以获取锁

### RAII 和 lock_guard

**RAII = Resource Acquisition Is Initialization（资源获取即初始化）**

**问题：**
如果在 `lock()` 和 `unlock()` 之间发生异常，锁可能不会被释放。

**解决方案：**
```cpp
#include <mutex>

std::mutex mtx;

void SafeFunction() {
    std::lock_guard<std::mutex> lock(mtx);  // 构造时自动加锁
    // 临界区代码
    // ...
    // 析构时自动解锁（即使发生异常也会解锁）
}
```

**函数解释：**
- `std::lock_guard<std::mutex>`：RAII 包装器类
- `lock(mtx)`：构造函数参数，传入要管理的互斥锁
- 构造时自动调用 `mtx.lock()`
- 析构时自动调用 `mtx.unlock()`

## 单例模式的实现方式

### 1. 懒汉式（线程不安全）

**特点：**
- 延迟创建：第一次使用时才创建实例
- 线程不安全：多线程环境下可能创建多个实例

```cpp
class LazySingleton {
private:
    static LazySingleton* instance;  // 静态指针，存储唯一实例
    
    // 私有构造函数
    LazySingleton() {
        std::cout << "LazySingleton 构造函数被调用" << std::endl;
    }
    
public:
    // 获取实例的静态方法
    static LazySingleton* GetInstance() {
        if (instance == nullptr) {          // 检查实例是否存在
            instance = new LazySingleton();  // 不存在则创建
        }
        return instance;                     // 返回实例指针
    }
    
    void DoSomething() {
        std::cout << "执行业务逻辑" << std::endl;
    }
};

// 静态成员变量需要在类外定义
LazySingleton* LazySingleton::instance = nullptr;
```

**代码详解：**
- `static LazySingleton* instance`：静态指针变量，属于类而不是对象
- `instance == nullptr`：检查指针是否为空（nullptr 表示空指针）
- `new LazySingleton()`：在堆上创建对象，返回指针
- `LazySingleton::instance = nullptr`：静态成员变量的定义和初始化

**使用示例：**
```cpp
int main() {
    // 第一次调用，会创建实例
    LazySingleton* s1 = LazySingleton::GetInstance();
    s1->DoSomething();
    
    // 第二次调用，返回同一个实例
    LazySingleton* s2 = LazySingleton::GetInstance();
    
    // 验证是否为同一个实例
    if (s1 == s2) {
        std::cout << "s1 和 s2 是同一个实例" << std::endl;
    }
    
    return 0;
}
```

**线程安全问题：**
```cpp
// 多线程环境下的问题
void ThreadFunction() {
    LazySingleton* instance = LazySingleton::GetInstance();
    // 可能得到不同的实例！
}

int main() {
    std::thread t1(ThreadFunction);  // 创建线程1
    std::thread t2(ThreadFunction);  // 创建线程2
    
    t1.join();  // 等待线程1结束
    t2.join();  // 等待线程2结束
    
    return 0;
}
```

**函数解释：**
- `std::thread t1(ThreadFunction)`：创建线程，参数是要执行的函数
- `t1.join()`：主线程等待 t1 线程执行完毕

### 2. 懒汉式（线程安全 - 双重检查锁定）

**特点：**
- 延迟创建
- 线程安全
- 性能较好（避免每次都加锁）

```cpp
#include <mutex>

class ThreadSafeLazySingleton {
private:
    static ThreadSafeLazySingleton* instance;  // 静态实例指针
    static std::mutex mtx;                     // 静态互斥锁
    
    ThreadSafeLazySingleton() {
        std::cout << "ThreadSafeLazySingleton 构造函数被调用" << std::endl;
    }
    
public:
    static ThreadSafeLazySingleton* GetInstance() {
        // 第一次检查（无锁）
        if (instance == nullptr) {
            std::lock_guard<std::mutex> lock(mtx);  // 加锁
            
            // 第二次检查（有锁）
            if (instance == nullptr) {
                instance = new ThreadSafeLazySingleton();
            }
        }
        return instance;
    }
    
    void DoSomething() {
        std::cout << "执行线程安全的业务逻辑" << std::endl;
    }
};

// 静态成员变量定义
ThreadSafeLazySingleton* ThreadSafeLazySingleton::instance = nullptr;
std::mutex ThreadSafeLazySingleton::mtx;
```

**双重检查锁定详解：**

**为什么需要两次检查？**

```cpp
// 只有一次检查的问题：
static Singleton* GetInstance() {
    std::lock_guard<std::mutex> lock(mtx);  // 每次都加锁
    if (instance == nullptr) {
        instance = new Singleton();
    }
    return instance;
}
// 问题：即使实例已经创建，每次调用都要加锁，性能差
```

```cpp
// 双重检查的优势：
static Singleton* GetInstance() {
    if (instance == nullptr) {              // 第一次检查：快速路径，无锁
        std::lock_guard<std::mutex> lock(mtx);  // 只在需要时加锁
        if (instance == nullptr) {          // 第二次检查：防止重复创建
            instance = new Singleton();
        }
    }
    return instance;
}
```

**执行流程分析：**

**情况1：实例已存在**
```cpp
Thread A: if (instance == nullptr)  // false，直接返回 instance
Thread B: if (instance == nullptr)  // false，直接返回 instance
// 无需加锁，性能好
```

**情况2：实例不存在，多线程同时访问**
```cpp
Thread A: if (instance == nullptr)     // true，进入锁区域
Thread B: if (instance == nullptr)     // true，等待锁

Thread A: lock(mtx)                    // 获得锁
Thread A: if (instance == nullptr)     // true，创建实例
Thread A: instance = new Singleton()   // 创建
Thread A: unlock(mtx)                  // 释放锁

Thread B: lock(mtx)                    // 获得锁
Thread B: if (instance == nullptr)     // false！实例已被A创建
Thread B: unlock(mtx)                  // 释放锁，返回已有实例
```

**代码注释详解：**
- `static std::mutex mtx`：静态互斥锁，所有实例共享
- `std::lock_guard<std::mutex> lock(mtx)`：RAII 锁管理器
- 第一次检查：避免不必要的加锁
- 第二次检查：防止多个线程同时通过第一次检查后重复创建

### 3. 饿汉式（线程安全）

**特点：**
- 程序启动时就创建实例
- 天然线程安全
- 可能浪费内存（即使不使用也会创建）

```cpp
class EagerSingleton {
private:
    // 静态实例，程序启动时就创建
    static EagerSingleton instance;
    
    EagerSingleton() {
        std::cout << "EagerSingleton 构造函数被调用" << std::endl;
    }
    
public:
    // 返回静态实例的引用
    static EagerSingleton& GetInstance() {
        return instance;  // 返回引用，不是指针
    }
    
    void DoSomething() {
        std::cout << "执行饿汉式业务逻辑" << std::endl;
    }
};

// 静态成员变量定义（程序启动时就会执行构造函数）
EagerSingleton EagerSingleton::instance;
```

**为什么线程安全？**

C++ 标准保证：
- 静态局部变量的初始化是线程安全的
- 全局/静态对象在 main 函数执行前就完成初始化
- 多个线程不会同时初始化同一个静态对象

**使用示例：**
```cpp
int main() {
    std::cout << "程序开始执行" << std::endl;
    
    // 获取实例（实例在程序启动时就已经创建了）
    EagerSingleton& s1 = EagerSingleton::GetInstance();
    s1.DoSomething();
    
    EagerSingleton& s2 = EagerSingleton::GetInstance();
    
    // 验证是否为同一个实例
    if (&s1 == &s2) {  // 比较地址
        std::cout << "s1 和 s2 是同一个实例" << std::endl;
    }
    
    return 0;
}
```

**代码解释：**
- `static EagerSingleton instance`：静态对象（不是指针）
- `EagerSingleton&`：返回引用类型
- `&s1 == &s2`：比较对象的地址（& 是取地址操作符）

### 4. 局部静态变量（推荐）

**特点：**
- C++11 标准保证线程安全
- 延迟创建
- 代码简洁
- 自动内存管理

```cpp
class LocalStaticSingleton {
private:
    LocalStaticSingleton() {
        std::cout << "LocalStaticSingleton 构造函数被调用" << std::endl;
    }
    
public:
    static LocalStaticSingleton& GetInstance() {
        static LocalStaticSingleton instance;  // 局部静态变量
        return instance;
    }
    
    void DoSomething() {
        std::cout << "执行局部静态变量业务逻辑" << std::endl;
    }
};
```

**为什么推荐这种方式？**

1. **线程安全**：C++11 保证静态局部变量初始化的线程安全性
2. **延迟创建**：第一次调用时才创建
3. **自动销毁**：程序结束时自动调用析构函数
4. **代码简洁**：无需手动管理锁和指针

**C++11 线程安全保证：**
```cpp
// 编译器会生成类似这样的代码：
static LocalStaticSingleton& GetInstance() {
    static std::once_flag flag;
    static alignas(LocalStaticSingleton) char storage[sizeof(LocalStaticSingleton)];
    static LocalStaticSingleton* instance = nullptr;
    
    std::call_once(flag, []() {
        instance = new(storage) LocalStaticSingleton();
    });
    
    return *instance;
}
```

**函数解释：**
- `std::once_flag`：确保某个操作只执行一次的标志
- `std::call_once(flag, lambda)`：确保 lambda 函数只被调用一次
- `alignas`：内存对齐说明符
- `new(storage)`：placement new，在指定内存位置构造对象

**多线程测试：**
```cpp
#include <thread>
#include <vector>

void TestThread(int threadId) {
    LocalStaticSingleton& instance = LocalStaticSingleton::GetInstance();
    std::cout << "线程 " << threadId << " 获取到实例地址: " 
              << &instance << std::endl;
}

int main() {
    std::vector<std::thread> threads;
    
    // 创建10个线程同时获取实例
    for (int i = 0; i < 10; ++i) {
        threads.emplace_back(TestThread, i);
    }
    
    // 等待所有线程完成
    for (auto& t : threads) {
        t.join();
    }
    
    return 0;
}
```

**代码解释：**
- `std::vector<std::thread>`：存储线程对象的容器
- `threads.emplace_back(TestThread, i)`：在容器末尾构造线程对象
- `emplace_back` vs `push_back`：emplace_back 直接构造，效率更高
- `auto& t`：自动推导类型的引用

### 5. 模板单例基类

**特点：**
- 可复用的单例模板
- 任何类都可以轻松变成单例
- 类型安全

```cpp
template<typename T>
class Singleton {
protected:
    Singleton() = default;                          // 默认构造函数
    virtual ~Singleton() = default;                 // 虚析构函数
    
public:
    // 禁用拷贝构造和赋值操作
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;
    Singleton(Singleton&&) = delete;               // 禁用移动构造
    Singleton& operator=(Singleton&&) = delete;    // 禁用移动赋值
    
    static T& GetInstance() {
        static T instance;  // 局部静态变量
        return instance;
    }
};

// 使用示例：让任何类变成单例
class GameManager : public Singleton<GameManager> {
    friend class Singleton<GameManager>;  // 允许基类访问私有构造函数
    
private:
    GameManager() {
        std::cout << "GameManager 构造函数" << std::endl;
    }
    
public:
    void StartGame() {
        std::cout << "游戏开始" << std::endl;
    }
    
    void EndGame() {
        std::cout << "游戏结束" << std::endl;
    }
};

class AudioManager : public Singleton<AudioManager> {
    friend class Singleton<AudioManager>;
    
private:
    AudioManager() {
        std::cout << "AudioManager 构造函数" << std::endl;
    }
    
public:
    void PlaySound(const std::string& soundName) {
        std::cout << "播放声音: " << soundName << std::endl;
    }
};
```

**模板语法解释：**
- `template<typename T>`：定义模板，T 是类型参数
- `Singleton<GameManager>`：模板实例化，T = GameManager
- `friend class Singleton<GameManager>`：友元声明，允许基类访问私有成员
- `= default`：使用编译器生成的默认实现
- `= delete`：明确禁用某个函数

**使用示例：**
```cpp
int main() {
    // 获取游戏管理器实例
    GameManager& gameManager = GameManager::GetInstance();
    gameManager.StartGame();
    
    // 获取音频管理器实例
    AudioManager& audioManager = AudioManager::GetInstance();
    audioManager.PlaySound("background_music.mp3");
    
    // 验证单例性
    GameManager& gameManager2 = GameManager::GetInstance();
    if (&gameManager == &gameManager2) {
        std::cout << "确实是同一个实例" << std::endl;
    }
    
    return 0;
}
```

## 线程安全深入分析

### 内存模型和可见性

**问题：**
即使使用了双重检查锁定，在某些情况下仍可能出现问题。

```cpp
// 潜在问题的代码
static Singleton* GetInstance() {
    if (instance == nullptr) {                    // 读操作
        std::lock_guard<std::mutex> lock(mtx);
        if (instance == nullptr) {
            instance = new Singleton();           // 写操作
        }
    }
    return instance;                              // 读操作
}
```

**内存重排序问题：**

`new Singleton()` 实际上包含三个步骤：
1. 分配内存
2. 调用构造函数
3. 将地址赋值给 instance

编译器可能会重排序为：
1. 分配内存
2. 将地址赋值给 instance（此时对象还未构造完成！）
3. 调用构造函数

**问题场景：**
```cpp
Thread A: instance = new Singleton();  // 步骤1、2完成，步骤3还未完成
Thread B: if (instance == nullptr)     // false！但对象还未构造完成
Thread B: return instance;             // 返回未完全构造的对象！
```

**解决方案：使用 std::atomic**

```cpp
#include <atomic>
#include <mutex>

class AtomicSingleton {
private:
    static std::atomic<AtomicSingleton*> instance;  // 原子指针
    static std::mutex mtx;
    
    AtomicSingleton() {
        std::cout << "AtomicSingleton 构造函数" << std::endl;
    }
    
public:
    static AtomicSingleton* GetInstance() {
        AtomicSingleton* tmp = instance.load(std::memory_order_acquire);
        
        if (tmp == nullptr) {
            std::lock_guard<std::mutex> lock(mtx);
            tmp = instance.load(std::memory_order_relaxed);
            
            if (tmp == nullptr) {
                tmp = new AtomicSingleton();
                instance.store(tmp, std::memory_order_release);
            }
        }
        
        return tmp;
    }
};

// 静态成员定义
std::atomic<AtomicSingleton*> AtomicSingleton::instance{nullptr};
std::mutex AtomicSingleton::mtx;
```

**原子操作和内存序解释：**

**std::atomic：**
- 原子类型，保证操作的原子性
- 读写操作不会被中断
- 提供内存序控制

**内存序（Memory Order）：**

1. **memory_order_acquire**：
   - 用于读操作
   - 保证在此操作之后的所有内存操作不会被重排序到此操作之前
   
2. **memory_order_release**：
   - 用于写操作
   - 保证在此操作之前的所有内存操作不会被重排序到此操作之后
   
3. **memory_order_relaxed**：
   - 最宽松的内存序
   - 只保证操作的原子性，不保证顺序

**函数详解：**
- `instance.load(std::memory_order_acquire)`：原子读取，获取语义
- `instance.store(tmp, std::memory_order_release)`：原子写入，释放语义
- `{nullptr}`：C++11 统一初始化语法

**为什么这样能解决问题？**

```cpp
Thread A: tmp = new Singleton();                           // 1. 分配+构造
Thread A: instance.store(tmp, memory_order_release);       // 2. 原子写入

Thread B: tmp = instance.load(memory_order_acquire);       // 3. 原子读取
```

`memory_order_release` 和 `memory_order_acquire` 配对使用，保证：
- Thread A 的步骤1完全完成后，才执行步骤2
- Thread B 的步骤3看到非空值时，Thread A 的步骤1一定已经完成

### std::call_once 实现

**最简洁的线程安全实现：**

```cpp
#include <mutex>

class CallOnceSingleton {
private:
    static CallOnceSingleton* instance;
    static std::once_flag flag;
    
    CallOnceSingleton() {
        std::cout << "CallOnceSingleton 构造函数" << std::endl;
    }
    
    // 初始化函数
    static void InitInstance() {
        instance = new CallOnceSingleton();
    }
    
public:
    static CallOnceSingleton* GetInstance() {
        std::call_once(flag, InitInstance);  // 确保 InitInstance 只被调用一次
        return instance;
    }
};

// 静态成员定义
CallOnceSingleton* CallOnceSingleton::instance = nullptr;
std::once_flag CallOnceSingleton::flag;
```

**std::call_once 详解：**

**函数签名：**
```cpp
template<class Callable, class... Args>
void call_once(std::once_flag& flag, Callable&& f, Args&&... args);
```

**参数解释：**
- `std::once_flag& flag`：标志对象，记录函数是否已被调用
- `Callable&& f`：要调用的函数（可以是函数指针、lambda、函数对象等）
- `Args&&... args`：传递给函数的参数（可变参数模板）

**工作原理：**
1. 检查 flag 是否已被设置
2. 如果未设置，执行函数 f，并设置 flag
3. 如果已设置，直接返回
4. 多个线程同时调用时，只有一个线程会执行函数，其他线程等待

**使用 lambda 表达式：**
```cpp
class LambdaSingleton {
private:
    static LambdaSingleton* instance;
    static std::once_flag flag;
    
public:
    static LambdaSingleton* GetInstance() {
        std::call_once(flag, []() {
            instance = new LambdaSingleton();
        });
        return instance;
    }
};
```

**lambda 表达式解释：**
- `[]() { ... }`：lambda 表达式语法
- `[]`：捕获列表（这里为空，不捕获外部变量）
- `()`：参数列表（这里为空）
- `{ ... }`：函数体

### 性能测试代码

```cpp
#include <chrono>
#include <thread>
#include <vector>

// 性能测试函数
template<typename SingletonType>
void PerformanceTest(const std::string& name, int threadCount, int callsPerThread) {
    auto start = std::chrono::high_resolution_clock::now();
    
    std::vector<std::thread> threads;
    
    for (int i = 0; i < threadCount; ++i) {
        threads.emplace_back([callsPerThread]() {
            for (int j = 0; j < callsPerThread; ++j) {
                auto& instance = SingletonType::GetInstance();

    for (auto& t : threads) {
        t.join();
    }
    
    auto end = std::chrono::high_resolution_clock::now();
    auto duration = std::chrono::duration_cast<std::chrono::microseconds>(end - start);
```

### 2. 资源管理器

```cpp
#include <unordered_map>
#include <memory>

class ResourceManager : public Singleton<ResourceManager> {
    friend class Singleton<ResourceManager>;
    
private:
    std::unordered_map<std::string, std::shared_ptr<void>> resources;
    
    ResourceManager() = default;
    
public:
    template<typename T>
    std::shared_ptr<T> LoadResource(const std::string& path) {
        auto it = resources.find(path);
        if (it != resources.end()) {
            // 资源已存在，直接返回
            return std::static_pointer_cast<T>(it->second);
        }
        
        // 加载新资源（这里简化为创建对象）
        auto resource = std::make_shared<T>();
        resources[path] = resource;
        
        std::cout << "加载资源: " << path << std::endl;
        return resource;
    }
    
    void UnloadResource(const std::string& path) {
        auto it = resources.find(path);
        if (it != resources.end()) {
            resources.erase(it);
            std::cout << "卸载资源: " << path << std::endl;
        }
    }
    
    size_t GetResourceCount() const {
        return resources.size();
    }
};
```

**代码解释：**
- `std::unordered_map`：哈希表，快速查找
- `std::shared_ptr<void>`：智能指针，自动内存管理
- `std::static_pointer_cast<T>`：智能指针类型转换
- `std::make_shared<T>()`：创建智能指针的推荐方式

### 3. 配置管理器

```cpp
#include <fstream>
#include <sstream>

class ConfigManager : public Singleton<ConfigManager> {
    friend class Singleton<ConfigManager>;
    
private:
    std::unordered_map<std::string, std::string> configs;
    
    ConfigManager() {
        LoadConfig("config.txt");
    }
    
    void LoadConfig(const std::string& filename) {
        std::ifstream file(filename);
        std::string line;
        
        while (std::getline(file, line)) {
            size_t pos = line.find('=');
            if (pos != std::string::npos) {
                std::string key = line.substr(0, pos);
                std::string value = line.substr(pos + 1);
                configs[key] = value;
            }
        }
        
        std::cout << "加载配置文件: " << filename << std::endl;
    }
    
public:
    std::string GetString(const std::string& key, const std::string& defaultValue = "") {
        auto it = configs.find(key);
        return (it != configs.end()) ? it->second : defaultValue;
    }
    
    int GetInt(const std::string& key, int defaultValue = 0) {
        std::string value = GetString(key);
        if (value.empty()) return defaultValue;
        
        try {
            return std::stoi(value);
        } catch (const std::exception&) {
            return defaultValue;
        }
    }
    
    float GetFloat(const std::string& key, float defaultValue = 0.0f) {
        std::string value = GetString(key);
        if (value.empty()) return defaultValue;
        
        try {
            return std::stof(value);
        } catch (const std::exception&) {
            return defaultValue;
        }
    }
    
    void SetConfig(const std::string& key, const std::string& value) {
        configs[key] = value;
    }
};
```

## 注意事项和最佳实践

### 1. 避免过度使用

**问题：**
不是所有类都需要是单例，过度使用会导致：
- 代码耦合度高
- 难以测试
- 难以扩展

**何时使用单例：**
- 确实只需要一个实例
- 需要全局访问
- 状态需要在整个程序中保持一致

**何时不用单例：**
- 可能需要多个实例
- 主要用于数据传递
- 无状态的工具类

### 2. 内存管理

**问题：**
使用 `new` 创建的单例可能不会被销毁。

**解决方案：**
```cpp
class SafeSingleton {
private:
    static SafeSingleton* instance;
    static std::mutex mtx;
    
    SafeSingleton() = default;
    
    // 内部删除器类
    class Deleter {
    public:
        ~Deleter() {
            if (SafeSingleton::instance) {
                delete SafeSingleton::instance;
                SafeSingleton::instance = nullptr;
            }
        }
    };
    
    static Deleter deleter;  // 静态删除器对象
    
public:
    static SafeSingleton* GetInstance() {
        if (instance == nullptr) {
            std::lock_guard<std::mutex> lock(mtx);
            if (instance == nullptr) {
                instance = new SafeSingleton();
            }
        }
        return instance;
    }
};

SafeSingleton* SafeSingleton::instance = nullptr;
std::mutex SafeSingleton::mtx;
SafeSingleton::Deleter SafeSingleton::deleter;  // 程序结束时自动调用析构函数
```

**更好的方案：使用局部静态变量**
```cpp
class BetterSingleton {
public:
    static BetterSingleton& GetInstance() {
        static BetterSingleton instance;  // 程序结束时自动销毁
        return instance;
    }
};
```

### 3. 单例的销毁顺序

**问题：**
多个单例之间可能存在依赖关系，销毁顺序很重要。

**解决方案：**
```cpp
// 使用智能指针管理单例
class ManagedSingleton {
private:
    static std::unique_ptr<ManagedSingleton> instance;
    static std::once_flag flag;
    
public:
    static ManagedSingleton& GetInstance() {
        std::call_once(flag, []() {
            instance = std::make_unique<ManagedSingleton>();
        });
        return *instance;
    }
    
    // 显式销毁方法
    static void Destroy() {
        instance.reset();
    }
};
```

## 总结

### 推荐实现方式

**C++11 及以后（推荐）：**
```cpp
class Singleton {
public:
    static Singleton& GetInstance() {
        static Singleton instance;
        return instance;
    }
    
private:
    Singleton() = default;
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;
};
```

**需要更多控制时：**
```cpp
class Singleton {
private:
    static std::unique_ptr<Singleton> instance;
    static std::once_flag flag;
    
public:
    static Singleton& GetInstance() {
        std::call_once(flag, []() {
            instance = std::make_unique<Singleton>();
        });
        return *instance;
    }
};
```

### 关键要点

1. **线程安全**：现代 C++ 推荐使用局部静态变量
2. **内存管理**：避免使用裸指针，优先使用栈对象或智能指针
3. **适度使用**：不要滥用单例模式
4. **测试友好**：考虑依赖注入等替代方案
