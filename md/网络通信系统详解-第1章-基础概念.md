## 网络架构概览

### 这个游戏是怎么联网的？

想象一下你和朋友一起玩游戏的场景：

```
你的电脑（客户端）          朋友的电脑（客户端）
      ↓                           ↓
      └────────→ 游戏服务器 ←──────┘
                    ↓
              协调器（管理员）
```

**简单来说：**
1. **客户端** - 就是你玩游戏的电脑，负责显示画面、接收你的操作
2. **游戏服务器** - 一台专门的电脑，负责运行游戏逻辑，确保大家看到的游戏状态一致
3. **协调器** - 一个小程序，负责创建和管理游戏服务器

## 核心概念解释

### 1. 什么是"专用服务器"（Dedicated Server）？

**通俗解释：**
- 就像打麻将，需要一个人专门负责洗牌、发牌、记分
- 这个人不参与游戏，只负责维护游戏规则
- 在我们的游戏里，服务器就是这个"裁判"

**为什么要用专用服务器？**
- ✅ 防作弊：所有重要计算都在服务器上，玩家改不了
- ✅ 公平性：大家看到的游戏状态都是服务器说了算
- ✅ 稳定性：即使有玩家掉线，游戏还能继续

### 2. 什么是 EOS（Epic Online Services）？

**通俗解释：**
- Epic 公司提供的一套免费的在线服务工具
- 就像微信提供的登录、聊天功能，但这是给游戏用的

**EOS 提供了什么？**
- 🔐 账号登录：玩家可以用 Epic 账号登录
- 🎮 会话管理：帮你找到其他玩家创建的游戏房间
- 🗣️ 语音聊天：玩家可以语音交流
- 🌐 跨平台：Windows、Linux 都能一起玩

### 3. 什么是"会话"（Session）？

**通俗解释：**
- 会话 = 一局游戏
- 就像你开了一个房间，等朋友进来一起玩

**会话包含什么信息？**
```cpp
会话名称：比如 "赵云飞的房间"
会话ID：一个唯一的编号，用来找到这个房间
端口号：服务器的"门牌号"，比如 7777
玩家列表：谁在这个房间里
```

### 4. 什么是"端口"（Port）？

**通俗解释：**
- 一台电脑可以同时运行很多程序
- 端口就像门牌号，告诉数据该去哪个程序

**举例：**
```
你的电脑 IP：192.168.1.100
  ├─ 端口 80：网页服务
  ├─ 端口 7777：游戏服务器1
  ├─ 端口 7778：游戏服务器2
  └─ 端口 7779：游戏服务器3
```

## 技术栈介绍

### 客户端（你玩游戏的部分）

| 技术 | 作用 | 通俗解释 |
|------|------|----------|
| Unreal Engine 5.5 | 游戏引擎 | 就像 Unity，用来做游戏的工具 |
| C++ | 编程语言 | 写游戏逻辑的语言 |
| EOS SDK | 在线服务 | Epic 提供的联网工具包 |

### 服务器端（游戏逻辑运行的地方）

| 技术 | 作用 | 通俗解释 |
|------|------|----------|
| Unreal Dedicated Server | 专用服务器 | 不显示画面，只跑游戏逻辑 |
| Docker | 容器化 | 把服务器打包，方便部署 |

### 协调器（管理服务器的小程序）

| 技术 | 作用 | 通俗解释 |
|------|------|----------|
| Python Flask | Web 框架 | 用 Python 写的小型网站 |
| HTTP API | 接口 | 客户端通过网址请求创建服务器 |

## 工作流程图

### 完整的联机流程

```
第1步：玩家登录
┌─────────────┐
│  玩家打开游戏  │
└──────┬──────┘
       ↓
┌─────────────┐
│ 用 Epic 账号登录 │  ← 使用 EOS 登录服务
└──────┬──────┘
       ↓
┌─────────────┐
│  登录成功！   │
└─────────────┘

第2步：创建/加入游戏
┌─────────────┐
│ 玩家点击"创建游戏" │
└──────┬──────┘
       ↓
┌─────────────────────┐
│ 客户端向协调器发送请求  │  ← HTTP 请求
│ "我要创建一个房间！"    │
└──────┬──────────────┘
       ↓
┌─────────────────────┐
│ 协调器启动游戏服务器   │  ← Python 脚本
│ 分配端口：7777        │
└──────┬──────────────┘
       ↓
┌─────────────────────┐
│ 服务器创建 EOS 会话   │  ← 注册到 EOS
│ 其他玩家可以搜索到    │
└──────┬──────────────┘
       ↓
┌─────────────────────┐
│ 客户端搜索并加入会话  │  ← 通过 EOS 查找
└──────┬──────────────┘
       ↓
┌─────────────┐
│  开始游戏！   │
└─────────────┘

第3步：游戏进行中
┌─────────────┐
│ 玩家按下技能键  │
└──────┬──────┘
       ↓
┌─────────────────────┐
│ 客户端发送到服务器    │  ← RPC 调用
│ "我要释放技能！"      │
└──────┬──────────────┘
       ↓
┌─────────────────────┐
│ 服务器验证并执行      │  ← 服务器权威
│ "OK，扣除法力值"      │
└──────┬──────────────┘
       ↓
┌─────────────────────┐
│ 服务器同步给所有玩家  │  ← 属性复制
│ "赵云飞释放了技能"    │
└──────┬──────────────┘
       ↓
┌─────────────┐
│ 所有玩家看到效果 │
└─────────────┘
```
