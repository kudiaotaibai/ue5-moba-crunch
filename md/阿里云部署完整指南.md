## 购买阿里云服务器

### 步骤1：登录阿里云

```
1. 访问：https://www.aliyun.com
2. 点击右上角"登录"
3. 使用支付宝或手机号登录
```

### 步骤2：进入 ECS 控制台

```
1. 点击"产品" → "云服务器 ECS"
2. 或直接访问：https://ecs.console.aliyun.com
```

### 步骤3：创建实例

**点击"创建实例"，按以下配置选择：**

#### 基础配置

```
计费方式：按量付费（测试用）或 包年包月（正式用）
地域：华东1（杭州）或 华北2（北京）
实例规格：
  - 入门：2核4GB（ecs.t6-c1m2.large）
  - 推荐：4核8GB（ecs.c6.xlarge）
  - 高配：8核16GB（ecs.c6.2xlarge）
```

**推荐配置（4核8GB）：**
```
CPU：4 核
内存：8 GB
适用场景：
- 5-10 个游戏房间
- 50-100 个玩家同时在线
```

#### 镜像选择

```
镜像类型：公共镜像
操作系统：Ubuntu 22.04 64位
```

**为什么选择 Ubuntu？**
```
✅ 免费开源
✅ 软件包丰富
✅ 社区支持好
✅ Docker 支持完善
```

#### 存储配置

```
系统盘：
  - 类型：ESSD云盘
  - 容量：40 GB（最小）
  - 推荐：80 GB（游戏服务器文件较大）

数据盘：
  - 可选，用于存储日志和数据
  - 推荐：100 GB
```

#### 网络配置

```
网络：专有网络（VPC）
公网IP：分配公网IPv4地址
带宽计费方式：按使用流量
峰值带宽：5 Mbps（可以后续调整）
```

**带宽说明：**
```
1 Mbps = 128 KB/s
5 Mbps = 640 KB/s
10 Mbps = 1.28 MB/s

推荐：
- 测试阶段：5 Mbps
- 正式运营：10-20 Mbps
```

#### 安全组配置

```
安全组：新建安全组
规则：
  - SSH（22）：允许
  - HTTP（80）：允许
  - HTTPS（443）：允许
  - 自定义TCP（7777-7800）：允许（游戏端口）
```

#### 系统配置

```
登录凭证：自定义密码
  - 设置一个强密码（包含大小写字母、数字、符号）
  - 记住这个密码！

实例名称：crunch-game-server
主机名：crunch-server
```

### 步骤4：确认订单

```
1. 检查配置
2. 勾选"《云服务器ECS服务条款》"
3. 点击"立即购买"
4. 支付
```

```bash

systemctl enable docker

docker --version
```

```bash

curl -L "https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

```
Docker Compose version v2.23.0
```

```bash

    "https://mirror.ccs.tencentyun.com",
    "https://docker.mirrors.ustc.edu.cn"

}

systemctl daemon-reload
```

## 修复配置问题

### 问题1：修复端口配置

**编辑 docker-compose.yaml：**

```bash
cd /root/ServerDeploy
vim docker-compose.yaml
```

**修改内容：**
```yaml
version: '3.8'

services:
  server:
    build: ./server
    image: crunch-server:latest

  coordinator:
    build: ./coordinator
    image: crunch-coordinator:latest
    container_name: crunch-coordinator
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    ports:
      - '7777:7777'  # 改成 7777
    networks:
      - crunch-network
    restart: unless-stopped

networks:
  crunch-network:
    driver: bridge
```

### 问题2：修复 Server Dockerfile

**编辑 server/Dockerfile：**

```bash
vim server/Dockerfile
```

**修改内容：**
```dockerfile
FROM ubuntu:22.04

# 安装依赖
RUN apt-get update && apt-get install -y \
    libicu70 \
    libssl3 \
    libcurl4 \
    && rm -rf /var/lib/apt/lists/*

# 复制服务器文件
COPY ./server/Crunch /app/Crunch
WORKDIR /app

# 设置执行权限
RUN chmod +x /app/Crunch/Binaries/Linux/CrunchServer

# 暴露端口
EXPOSE 7777

# 启动命令
ENTRYPOINT ["/app/Crunch/Binaries/Linux/CrunchServer", "Crunch", "-server", "-log"]
```

### 问题3：修复 Coordinator 代码

**编辑 coordinator/coordinator/coordinator.py：**

```bash
vim coordinator/coordinator/coordinator.py
```

**修改启动端口：**
```python
if __name__ == "__main__":
    app.run(host="0.0.0.0", port=7777)  # 改成 7777
```

**修改服务器启动命令（使用 Docker）：**
```python
import docker

docker_client = docker.from_env()

def CreateServerDocker(sessionName, sessionSearchId):
    global nextAvailablePort
    
    # 使用 Docker 创建容器
    container = docker_client.containers.run(
        image='crunch-server:latest',
        detach=True,
        ports={'7777/tcp': nextAvailablePort},
        environment={
            'SESSION_NAME': sessionName,
            'SESSION_SEARCH_ID': sessionSearchId,
            'PORT': '7777'
        },
        name=f'game-server-{sessionSearchId}',
        network='serverdeploy_crunch-network'
    )
    
    usedPort = nextAvailablePort
    nextAvailablePort += 1
    
    return usedPort
```

## 配置防火墙

### 阿里云安全组配置

**在 ECS 控制台：**

```
1. 进入实例详情
2. 点击"安全组"标签
3. 点击安全组ID
4. 点击"配置规则"
5. 点击"添加安全组规则"
```

**添加以下规则：**

| 规则方向 | 协议类型 | 端口范围 | 授权对象 | 描述 |
|---------|---------|---------|---------|------|
| 入方向 | TCP | 22 | 0.0.0.0/0 | SSH |
| 入方向 | TCP | 7777 | 0.0.0.0/0 | 协调器 |
| 入方向 | TCP | 7778-7800 | 0.0.0.0/0 | 游戏服务器 |
| 入方向 | UDP | 7778-7800 | 0.0.0.0/0 | 游戏服务器 |

### 服务器防火墙配置

```bash
# Ubuntu 默认使用 ufw
# 允许 SSH
ufw allow 22/tcp

# 允许协调器端口
ufw allow 7777/tcp

# 允许游戏服务器端口
ufw allow 7778:7800/tcp
ufw allow 7778:7800/udp

# 启用防火墙
ufw enable

# 查看状态
ufw status
```

## 域名和SSL配置

### 步骤1：购买域名（可选）

**等待生效：约 10 分钟**

### 步骤3：申请免费 SSL 证书

```
1. 访问：https://yundun.console.aliyun.com/?p=cas
2. 点击"购买证书"
3. 选择"免费证书"
4. 购买（¥0）
5. 申请证书
6. 填写域名：crunch-game.com
7. 验证域名所有权
8. 下载证书
```

### 步骤4：配置 Nginx（HTTPS）

```bash
# 安装 Nginx
apt install -y nginx

# 上传证书
mkdir -p /etc/nginx/ssl
# 上传证书文件到 /etc/nginx/ssl/

# 配置 Nginx
vim /etc/nginx/sites-available/crunch
```

**Nginx 配置：**
```nginx
server {
    listen 80;
    server_name crunch-game.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name crunch-game.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    location / {
        proxy_pass http://localhost:7777;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**启用配置：**
```bash
ln -s /etc/nginx/sites-available/crunch /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

## 常见问题

### Q1：连接超时怎么办？

**A：检查防火墙和安全组**

```bash
# 1. 检查服务器防火墙
ufw status

# 2. 检查阿里云安全组
# 在 ECS 控制台查看

# 3. 检查服务是否运行
docker ps

# 4. 检查端口监听
netstat -tlnp | grep 7777
```

### Q2：容器启动失败怎么办？

**A：查看日志**

```bash
# 查看容器日志
docker logs <容器ID>

# 查看详细信息
docker inspect <容器ID>

# 重新构建镜像
docker-compose build --no-cache
```

### Q3：内存不足怎么办？

**A：增加 Swap 或升级配置**

```bash
# 创建 Swap（临时方案）
fallocate -l 4G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' >> /etc/fstab

# 或者升级服务器配置
# 在阿里云控制台升级实例规格
```

## 成本优化

### 按量付费 vs 包年包月

| 计费方式 | 优点 | 缺点 | 适用场景 |
|---------|------|------|----------|
| 按量付费 | 灵活，随时释放 | 价格较高 | 测试、短期使用 |
| 包年包月 | 价格便宜（约7折） | 需要长期使用 | 正式运营 |

### 带宽优化

```
按固定带宽：¥23/Mbps/月
按使用流量：¥0.8/GB

建议：
- 测试阶段：按流量计费
- 正式运营：按固定带宽（如果流量大）
```

### 使用抢占式实例

```
抢占式实例 = 价格便宜（约1折）
但可能被回收（提前3分钟通知）

适用场景：
- 测试环境
- 可以随时重启的服务
```
